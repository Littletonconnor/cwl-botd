================================================================================
CWL-BOTD: ENTERPRISE-READY BOT DETECTION LIBRARY — IMPLEMENTATION PLAN
================================================================================

Research basis: FingerprintJS BotD, CreepJS, puppeteer-extra-plugin-stealth
evasion techniques, Cloudflare Bot Management signals, DataDome/HUMAN Security
approaches, isbot UA parsing, Apify fingerprint-injector countermeasures.

Current state: ~35% complete. Collectors exist but detectors are empty stubs.
No tests, no scoring engine, no classification output, no build pipeline for
the bot-detection package.

PHASE 2: CORE DETECTION ENGINE
================================================================================
Priority: CRITICAL — This is the heart of the library (currently 0% done).

[ ] 2.1  Design detector interface and registry
         - Define Detector interface: { name, category, detect(data) => Signal }
         - Define Signal type: { detected: boolean, score: number, reason: string }
         - Create DetectorRegistry class for registering/running detectors
         - Support detector categories: automation, inconsistency, behavioral
         - Allow user-registered custom detectors (plugin system)

[ ] 2.2  Implement automation artifact detectors
         Based on FingerprintJS BotD's proven detection techniques:
         - WebDriver detector: navigator.webdriver === true
         - Headless Chrome flags: missing chrome.runtime, chrome.loadTimes
         - HeadlessChrome UA substring detection
         - window.chrome object presence/absence analysis
         - CDP (Chrome DevTools Protocol) runtime detection
         - Permissions API anomaly: Notification permission in headless
         - Navigator properties count (headless has fewer)
         - Missing plugin array (headless typically has 0 plugins)
         - Connection rtt === 0 (headless network anomaly)
         - DeviceMemory === 0 or missing
         - HardwareConcurrency anomalies

[ ] 2.3  Implement tool-specific detectors
         Target each automation framework's unique artifacts:

         Selenium/WebDriver:
         - document.$cdc_asdjflasutopfhvcZLmcfl_ (ChromeDriver leak)
         - window.__selenium_unwrapped, __webdriver_evaluate, __driver_evaluate
         - document.__webdriver_script_fn, __selenium_evaluate
         - window.callSelenium, _selenium, callPhantom

         Puppeteer:
         - window.__puppeteer_evaluation_script__
         - Headless UA detection (HeadlessChrome)
         - Missing window.chrome.app, window.chrome.csi

         Playwright:
         - window.__playwright, __pw_manual
         - playwright-specific evaluation markers

         PhantomJS:
         - window.callPhantom, window._phantom
         - PhantomJS UA string

         Nightmare/Electron:
         - window.__nightmare, process.versions.electron
         - window.require presence in browser context

[ ] 2.4  Implement browser environment detectors
         Signals that distinguish real browsers from spoofed environments:
         - eval.toString().length check (varies by engine: V8=33, SpiderMonkey=37)
         - Error stack trace format analysis (engine-specific patterns)
         - Function.prototype.toString native code check
         - Proxy/getter trap detection on navigator properties
         - Object.getOwnPropertyDescriptor checks on overridden props
         - iframe contentWindow consistency checks
         - Performance.now() precision analysis (reduced in automation)
         - Date.now() vs performance.now() clock skew
         - Screen dimensions vs window dimensions consistency

[ ] 2.5  Implement scoring engine
         - Define score weights per detector (configurable)
         - Implement weighted scoring aggregation
         - Define threshold levels: definiteBot (>0.9), likelyBot (>0.7),
           suspicious (>0.4), likelyHuman (<0.4)
         - Support custom threshold configuration
         - Return structured result:
           {
             bot: boolean,
             botKind: 'headless' | 'selenium' | 'puppeteer' | 'playwright' |
                      'phantomjs' | 'unknown' | 'none',
             confidence: number,    // 0-1
             reasons: string[],     // human-readable explanations
             score: number,         // raw weighted score
             signals: Signal[],     // individual detector results
           }

================================================================================
ARCHITECTURAL PRINCIPLES
================================================================================

1. ZERO DEPENDENCIES — No runtime deps. Everything built from scratch.
2. TREE-SHAKEABLE — Every detector independently importable.
3. SSR-SAFE — Never access window/document without guards.
4. PRIVACY-FIRST — No PII collection. Optional fingerprinting disable.
5. PERFORMANCE — Sub-5ms collection, sub-2ms detection, <15KB gzipped.
6. EXTENSIBLE — Plugin system for custom detectors and collectors.
7. TYPE-SAFE — Full TypeScript with strict mode, exported types.
8. TRANSPARENT — Every detection decision includes human-readable reason.

================================================================================
KEY REFERENCE IMPLEMENTATIONS TO STUDY
================================================================================

1. FingerprintJS BotD (github.com/nicedayfor/BotD)
   - Their detector structure: one file per signal, each exports a detector fn
   - Their scoring: simple boolean per detector, aggregate to botKind
   - Key detectors: app_version, document_element_keys, error_trace,
     eval_length, function_bind, languages_inconsistency, mime_types_consistence,
     notification_permissions, plugins_array, process, product_sub, rtt,
     user_agent, webdriver, webgl, window_external, window_size

2. CreepJS (github.com/nicedayfor/creepjs)
   - Advanced techniques: lies detection (property descriptor analysis),
     trash detection, prototype chain analysis
   - Their approach to detecting Proxy-based property spoofing

3. puppeteer-extra-plugin-stealth
   - What it patches tells us what to detect:
     chrome.app, chrome.csi, chrome.loadTimes, chrome.runtime,
     iframe.contentWindow, media.codecs, navigator.hardwareConcurrency,
     navigator.languages, navigator.permissions, navigator.plugins,
     navigator.webdriver, sourceurl, user-agent-override, webgl.vendor,
     window.outerdimensions
